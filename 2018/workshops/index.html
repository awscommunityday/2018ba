Join the Lacework team on for a live demonstration of how behavioral analysis can be applied at scale for continuous security and compliance monitoring of your AWS infrastructure using Lacework. Vikram Kapoor, Co-Founder at Lacework will walk attendees through Lacework with a specific focus on how we automatically analyze AWS CloudTrail and AWS Config data to ensure that security best practices are in place and that data anomalies are detected to help prevent ransomware, Bitcoin mining, or container security issues. 

Automating AWS resource management can generate material savings of 20%-60% of monthly spend. Rolling your own automation code to address this can be tricky and not a good use of your valuable DevOps time. This workshop will explore using a managed SaaS policy engine, the GorillaStack Rules Engine, that makes it quick and easy to cost optimize through automated scheduling and cleanup of unutilized and under utilized resources.

Accelerating CI/CD + CS in AWS cloud environments.

In order to confidently scale your AWS deployments, continuous security must be built into your continuous integration and continuous delivery architecture. Participate in a series of interactive capture the flag challenges to get hands-on experience with DevSecOps. We’ll teach you how to think like a Security Ninja, highlight common mistakes that can have catastrophic consequences, and provide tips to avoid them.

In this workshop, we’ll talk about data-driven DevOps and how the cross-cutting metrics from dev, QA, and ops can be integrated to provide you increasing productivity, reducing maintenance efforts and promoting automated integration between the different parts of the pipeline. This workshop will explore using managed SaaS Artifactory instance to serve as a single access point organizing all of your binary resources including proprietary libraries, remote artifacts and other 3rd party resources.

At this workshop you will learn how to extend the capability of your AWS cloud with enterprise data storage functionality, including SAN and NAS, Active Directory, snapshots, replication, multi-zone high availability, online volume migration, thin provisioning, and more.

How to securely migrate to AWS.

Security concerns can hamper your AWS migration plans. Securing application communications across VPCs, Availability Zones, and Regions, requires complex, error-prone configurations. Moreover, deploying cloud-native applications with ECS, EKS, Fargate, and Lambda create further security concerns if you are using legacy methods for locking down your cloud environment.

In this workshop you will learn how to secure an application deployed on ECS and EKS in 3 easy steps without any changes to the application.
